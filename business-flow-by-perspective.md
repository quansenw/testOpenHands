# 北海365分类信息系统 — 前台用户视角业务流程

> 本文档从前台两类用户的视角出发，完整梳理分类信息的主业务流程。
> 两类用户并非系统中的固定角色，而是同一个用户在不同场景下的行为视角。

---

## 视角定义

| 视角 | 说明 | 核心诉求 |
|------|------|----------|
| **信息提供方** | 有东西要卖/有服务要推/有信息要发布的人 | 让目标受众看到我的信息，产生联系 |
| **信息求购方** | 要买东西/找服务/找信息的人 | 快速找到合适的信息，联系到提供方 |

> 同一个用户既可以发布二手手机（提供方），也可以搜索家政服务（求购方）。

---

# 一、信息提供方

> 核心目标：把信息发出去 → 被人看到 → 被人联系

## 1. 进入平台

### 1.1 注册与登录

```
打开 APP / PC / 小程序 → 点击"发布"或进入"我的"
  → 系统检测未登录 → 弹出登录页
    → 输入手机号 → 获取短信验证码（60s 倒计时）→ 输入验证码 → 登录成功
```

- 登录即注册，首次使用手机号自动创建账号
- 登录成功后返回 Token，后续请求通过 Header 携带
- 浏览首页、分类列表、信息详情无需登录；发布、评论、联系等操作需要登录

### 1.2 身份体系

信息提供方可选择两种身份发布信息：

| 身份 | 准入条件 | 特点 |
|------|---------|------|
| **个人** | 登录即可 | 适合偶尔发布、个人出售 |
| **推荐商家** | 需通过商家认证 | 有独立商家页、享受推荐展示、可购买套餐 |

---

## 2. 发布信息（核心流程）

这是信息提供方最重要的操作，是一个多步骤流程：

### 2.1 第1步：选择分类

```
点击"发布"按钮
  → 进入一级分类页（图标宫格）
    → 点击一级分类（如"跳蚤市场"）
      → 进入二级分类页（纵向列表）
        → 点击二级分类（如"二手手机"）
```

- 一级分类：商铺店面、二手车、跳蚤市场、生活服务、休闲娱乐、家政服务等
- 最终发布落点是二级叶子分类
- 不同分类对应不同的发布模板（字段组合不同）

### 2.2 第2步：选择发布身份

```
选择二级分类后 → 进入身份选择页（两张大卡片）
  ├─ 点击"个人" → 直接进入发布表单
  └─ 点击"推荐商家"
       ├─ 已认证且在有效期内 → 直接进入发布表单
       ├─ 未认证 → 跳转商家认证页（见第5节）
       ├─ 认证被拒 → 跳转重认证页，显示驳回原因
       └─ 认证过期 → 跳转续期认证页
```

### 2.3 第3步：填写发布表单

进入表单后，系统根据所选分类的模板动态生成字段：

**常见模板与字段组合**：

| 模板 | 典型字段 |
|------|---------|
| 价格突出（二手交易类） | 标题、地址、图片、价格、详细描述、标签、联系人、手机号 |
| 电话突出（服务类） | 标题、地址、图片、详细描述、标签、联系人、手机号 |
| 招聘模式 | 职位、公司、地址、薪资范围、薪资类型、简介、标签、联系人、手机号 |
| 拼车模式 | 出发地、目的地、出发时间、时间标记、详细描述、标签、联系人、手机号 |
| 图文分享 | 图片、详细描述、标签、联系人、手机号 |

**字段渲染规则**：
- 系统字段按顺序渲染
- 遇到"详细描述"字段时，插入运营配置的全部自定义字段
- 然后继续渲染剩余系统字段

**图片上传**：
- 最多 9 张图片
- 通过七牛云直传（前端获取上传 Token → 直传七牛 → 保存路径/宽高信息）

**必填校验**：
- 必填项缺失时阻断提交并定位提示
- 图片上传失败支持重试与删除

### 2.4 第4步：增值项配置（可选）

填写完表单后，页面下方展示增值项：

```
┌─────────────────────────────────────┐
│  置顶推广                            │
│  ├─ 分类首页+类别置顶               │
│  ├─ 时长选择：1天/3天/7天/15天...     │
│  └─ 对应价格实时更新                 │
├─────────────────────────────────────┤
│  推荐套餐                            │
│  ├─ 套餐卡片列表                     │
│  └─ 选择后自动计入合计               │
├─────────────────────────────────────┤
│  显示天数提示：如"显示365天 免费"     │
├─────────────────────────────────────┤
│  合计金额：¥XX.XX                    │
├─────────────────────────────────────┤
│  [ 发布 ]                            │
└─────────────────────────────────────┘
```

### 2.5 第5步：支付（如合计金额 > 0）

```
点击"发布" → 表单校验通过
  ├─ 合计为 0 → 跳过支付，直接进入成功页
  └─ 合计 > 0 → 进入支付页
       → 显示商品名称与金额
       → 选择支付方式（微信支付 / 支付宝）
       → 点击"确定支付" → 调起支付
         ├─ 支付成功 → 进入成功页
         └─ 支付失败/取消 → 可重新支付，不丢失订单
```

**费用说明**：
- 每月有免费发布额度（按分类×用户组配置），额度内免费
- 超额后按次收费
- 置顶按时长单独收费
- 支付介质：现金（RMB）、金币、iOS 虚拟币

### 2.6 第6步：发布成功

```
┌─────────────────────────────┐
│       ✓ 发布成功              │
│                              │
│   [ 再发一条 ]  [ 关闭 ]     │
└─────────────────────────────┘
```

- "再发一条"→ 进入新一轮发布流程
- "关闭"→ 返回首页
- 信息状态变为 **待审**，等待运营审核

## 3. 发布后的信息生命周期

信息提供方发布后，信息会经历以下状态流转：

```
发布提交
  │
  ▼
待审（等待运营审核）
  │
  ├──审核通过──→ 已发布（前台可见，求购方可浏览和联系）
  │                 │
  │                 ├── 到期 → 已失效（前台不再展示）
  │                 │            └── 可刷新激活 → 重新上线
  │                 │
  │                 └── 主动下架 → 已失效
  │
  └──审核拒绝──→ 已拒绝（显示拒绝理由）
                    └── 可编辑内容后重新提交
```

**关键规则**：
- 审核拒绝时，已支付的发布费/置顶费自动原路退回
- 已发布的信息被编辑后，若触发敏感词会自动重新进入待审
- 已发布的信息有有效期，到期后自动变为已失效

---

## 4. 管理我的信息（用户中心）

发布后，提供方在"我的"中管理自己的信息：

### 4.1 我的发布列表

```
底部 Tab"我的" → 我的发布 → 按状态分 Tab 浏览
```

| 状态 Tab | 含义 | 可执行操作 |
|----------|------|-----------|
| 全部 | 所有信息 | 查看 |
| 通过 | 已上线展示中 | 编辑、下架、刷新、续期 |
| 待审 | 等待运营审核 | 查看 |
| 拒绝 | 审核未通过 | 查看拒绝理由、编辑后重新提交 |
| 待支付 | 支付未完成 | 继续支付 |
| 已失效 | 已过期或已下架 | 刷新激活 |
| 已删除 | 已删除 | 查看 |

### 4.2 关键操作说明

| 操作 | 说明 |
|------|------|
| **刷新** | 让信息重新出现在列表顶部，相当于"顶一下"，可能消耗刷新次数或收费 |
| **续期** | 延长信息展示有效期 |
| **编辑** | 修改信息内容（通过态编辑触发敏感词会重新进入待审） |
| **下架** | 主动停止展示 |

### 4.3 消息/咨询列表

> 备注：咨询聊天接入第三方聊天工具。

```
我的 → 消息通知 → 咨询会话列表
  → 展示与求购方的聊天会话（按最近消息时间排序）
  → 点击某条会话 → 进入聊天详情页，继续沟通
```

### 4.4 我的评论

```
我的 → 我的评论 → 查看评论记录 → 点击可跳转到所属信息详情
```

## 5. 商家认证（升级身份，可选）

普通个人用户如果想获得更多曝光和专属商家页面，可申请商家认证：

### 5.1 认证入口

- 发布时选择"推荐商家"身份 → 系统检测未认证 → 跳转认证页
- 或在用户中心主动进入"我的认证"

### 5.2 认证流程

```
查看认证须知弹窗（会员说明 + 客服电话）→ 点击"知道了"
  → 填写商家资料 → 提交 → 等待运营审核
```

**需填写的商家资料**：

| 字段 | 说明 |
|------|------|
| 商家 Logo | 图片上传 |
| 商家名称 | 店铺显示名 |
| 所属分类 | 下拉选择 |
| 店铺特色标签 | 多标签 |
| 联系电话 | 手机号 |
| 企业地址 | 地址输入 |
| 企业名称 | 公司全称 |
| 企业介绍 | 详细描述 |

### 5.3 认证结果

```
提交 → 待审
  ├── 通过 → 获得认证商家身份（有效期通常365天）
  │         ├── 可以"推荐商家"身份发布
  │         ├── 获得独立商家详情页
  │         ├── 首页推荐商家区优先展示
  │         └── 分类列表中展示商家卡片
  │
  └── 拒绝 → 显示驳回原因 → 可编辑后重新提交
```

### 5.4 认证后的额外能力

| 能力 | 说明 |
|------|------|
| 商家详情页 | 独立页面展示商家 Logo、介绍、联系方式、发布的所有信息 |
| 推荐展示 | 首页"推荐商家"区、分类列表商家卡片优先展示 |
| 认证标识 | 信息卡片和详情页显示"认证商家"标签 |
| 可能的免审权 | 部分认证商家组可配置发布免审（`skip_verify=1`） |

---

## 6. 购买套餐（批量权益，可选）

提供方可购买套餐获得批量发布权益：

### 6.1 套餐内容

一个套餐是复合权益包，可能包含：

| 权益项 | 示例 |
|--------|------|
| 免费发布次数 | 30 次 |
| 刷新次数 | 10 次 |
| 类别置顶天数 | 7 天 |
| 分类首页+类别置顶天数 | 3 天 |
| 查看简历库次数 | 5 次（招聘类） |
| 自定义赠送 | 视具体套餐 |

### 6.2 购买流程

```
发布页"推荐套餐"入口 / 套餐购买页
  → 查看可用套餐列表（按所选分类过滤）
    → 选择套餐 → 确认价格
      → 支付（微信/支付宝）
        → 获得套餐权益
          → 后续发布时自动消耗套餐内权益
```

### 6.3 套餐规则

- 套餐按分类关联（全分类通用 / 指定分类专用）
- 权益有有效期，过期未使用的作废
- 部分套餐有限购次数
- iOS 端使用虚拟币价格，安卓/PC 端使用现金价格

---

## 7. 等待被联系

信息发布并通过审核后，信息提供方进入"等待被联系"阶段：

- 求购方在详情页点击"打电话" → 提供方收到来电
- 求购方在详情页点击"咨询" → 提供方收到消息（备注：咨询聊天是接入第三方聊天工具）
- 求购方在详情页发表评论 → 提供方可在"我的发布"的详情中看到评论

# 二、信息求购方

> 核心目标：找到需要的信息 → 看清楚内容 → 联系到提供方

## 1. 进入平台

### 1.1 无需登录即可浏览

求购方打开 APP / PC / 小程序后，可以直接浏览以下公共页面，无需登录：

- 首页（分类入口、推荐信息流、搜索）
- 分类列表页（某分类下的信息列表）
- 信息详情页（单条信息完整内容）
- 商家详情页（某商家的完整介绍）

**需要登录的操作**：联系提供方（打电话/咨询）、发表评论、收藏信息

### 1.2 登录（按需触发）

当求购方触发需要登录的操作时，系统弹出登录引导：

```
点击"打电话"/"咨询"/"评论" → 系统检测未登录
  → 弹出登录页 → 输入手机号 → 验证码 → 登录成功
    → 自动回到触发点继续操作
```

---

## 2. 发现信息（找到目标）

求购方有多条路径找到目标信息：

### 2.1 路径一：首页浏览

```
首页
  ├── 顶部搜索框 → 输入关键词 → 搜索结果
  ├── 分类图标宫格 → 点击分类 → 分类列表页
  ├── Banner 广告 → 点击 → 对应落地页
  ├── 推荐商家区 → 点击商家卡片 → 商家详情页
  └── 信息流卡片 → 点击 → 信息详情页
```

**首页区块顺序**（从上到下）：
1. 顶部标题与搜索
2. Banner 轮播
3. 分类图标宫格（一级分类入口）
4. 公告条
5. 推荐商家区
6. 最新发布 / 信息流

### 2.2 路径二：分类浏览

```
首页点击分类图标（或底部 Tab）
  → 分类列表页
    ├── 顶部：子分类切换（二级分类 Tab）
    ├── 筛选条件：地区、价格等
    ├── 排序方式
    ├── 信息卡片列表 → 点击 → 信息详情页
    └── 商家卡片列表 → 点击 → 商家详情页
```

**信息卡片展示要素**：
- 缩略图（左侧）
- 标题与摘要（右侧）
- 价格或核心字段
- 置顶角标（部分）
- 标签组（会员等级、服务标签等）
- 发布时间 / 相对时间

**分类列表不只是信息列表**：
- 部分分类同时展示信息卡片和商家卡片
- 商家卡片点击进入商家详情页，不是信息详情页

### 2.3 路径三：搜索

```
首页搜索框 → 输入关键词 → 搜索结果页
  → 按分类/条件筛选 → 点击结果卡片 → 信息详情页
```

### 2.4 路径四：推荐商家

```
首页"推荐商家"区 → 点击商家卡片 → 商家详情页
  → 浏览商家发布的信息列表 → 点击 → 信息详情页
```

---

## 3. 查看信息详情（看清楚内容）

点击信息卡片后，进入信息详情页：

### 3.1 页面结构（从上到下）

```
┌─────────────────────────────────────┐
│  ← 返回          页面标题            │
├─────────────────────────────────────┤
│                                     │
│  [ 主图展示区 ]                      │

│                                     │
├─────────────────────────────────────┤
│  标题                                │
│  ¥ 价格                             │
│  发布时间                            │
├─────────────────────────────────────┤
│                                     │
│  关键信息字段（地址、类型等）          │
│                                     │
│  详细描述                            │

│  ................................    │
│                                     │

│  [ 咨询 ]        [ 打电话 ]           │  ← 底部固定动作栏
└─────────────────────────────────────┘
```

### 3.2 图片浏览

- 点击主图进入大图浏览模式
- 多图支持左右切换

### 3.3 字段展示

- 关键信息字段（价格、地址、类型等）优先展示在首屏或接近首屏
- 详细描述支持长文本
- 自定义字段（如品牌、型号等）插入在描述区域上方

### 3.4 评论摘要区

- 默认展示少量评论预览
- 标题显示 `全部评论 · N`（N 为评论总数）
- 点击标题或"查看更多"进入全部评论页

## 4. 联系提供方（做出行动）

这是求购方最核心的目标动作：

### 4.1 打电话

```
点击底部"打电话"按钮
  ├── 已登录 → 调起系统拨号，号码为提供方手机号
  └── 未登录 → 弹出登录引导 → 登录后再调起拨号
```

### 4.2 咨询

> 备注：咨询是接入第三方聊天工具的。

```
点击底部"咨询按钮
  ├── 已登录 → 进入与提供方的聊天页面
  └── 未登录 → 弹出登录引导 → 登录后进入聊天
```

## 5. 互动与评论

求购方可以在信息详情页发表评论或提问：

### 5.1 发表评论

```
信息详情页 → 点击评论入口
  → 检查登录状态（未登录则先登录）
    → 弹出底部评论输入框
      → 输入内容（1-500字）→ 点击"发表"
        → 提示"评论已提交，审核后展示"
```

- 评论提交后为待审核状态，运营审核通过后才在前台展示
- 前端不承诺评论立刻上屏

### 5.2 回复评论

```
评论列表中 → 点击某条评论的"回复"按钮
  → 弹出输入框，标题显示"回复 {用户昵称}"
    → 输入内容 → 点击"发表"
      → 提示"评论已提交，审核后展示"
```

- 评论采用两级嵌套结构：一级评论 + 回复列表
- 回复的回复仍挂在同一个一级评论下（不无限嵌套）
- 回复中显示"回复 {被回复人昵称}"标签

### 5.3 查看全部评论

```
详情页评论摘要区 → 点击"全部评论 · N"
  → 进入全部评论页（详情页的二级页面）
    ├── 顶部导航与评论总数
    ├── 评论输入入口
    ├── 评论列表（一级评论 + 缩进回复）
    └── 加载更多 / 已到底
```

- 分页针对一级评论，每条一级评论下挂载全部已审核回复
- 返回后回到原详情页

# 三、两个视角的交汇点

信息提供方和求购方在以下环节产生交互：

```
┌──────────────┐                              ┌──────────────┐
│   信息提供方   │                              │   信息求购方   │
├──────────────┤                              ├──────────────┤
│              │    ① 信息展示                  │              │
│  发布信息 ───────────────────────────────────→  浏览信息     │
│              │                              │              │
│              │    ② 电话/咨询               │              │
│  接到联系 ←────────────────────────────────── 发起联系      │
│              │                              │              │
│              │    ③ 评论互动                  │              │
│  看到评论 ←────────────────────────────────── 发表评论      │
│              │                              │              │
│              │    ④ 商家浏览                  │              │
│  商家页展示 ──────────────────────────────────→ 浏览商家     │
│              │                              │              │
└──────────────┘                              └──────────────┘
```

| 交汇环节 | 提供方动作 | 求购方动作 | 系统参与 |
|---------|-----------|-----------|---------|
| 信息展示 | 发布并通过审核 | 浏览/搜索/筛选 | 运营审核上线 |
| 联系沟通 | 被动接听/回复 | 主动拨打/发消息 | 登录拦截、防骚扰 |
| 评论互动 | 查看评论反馈 | 发表评论/提问 | 评论审核后展示 |
| 商家信任 | 商家认证+维护商家页 | 浏览商家页评估可信度 | 认证审核 |
