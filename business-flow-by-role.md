# 北海365分类信息系统 — 按角色划分的主业务流程

> 本文档基于 `docs/modules/` 各模块设计文档整理，梳理四种角色在分类信息核心链路中的完整操作流程。

---

## 角色定义

| 角色 | 说明 | 系统标识 |
|------|------|----------|
| 普通用户 | 以"个人"身份使用平台的注册用户 | `s_user` 普通记录 |
| 商家 | 提交了商家认证但尚未通过（或已过期/被拒绝）的用户 | `s_group_user.status ≠ 1` |
| 认证商家 | 商家认证已通过且在有效期内的用户 | `s_group_user.group_id=1025, status=1` |
| 运营人员 | 后台管理端操作人员 | `s_admin_manager` |

---

## 一、普通用户

### 1.1 注册与登录

```
输入手机号 → 获取短信验证码（60s 倒计时）→ 输入验证码 → 登录/自动注册
```

- **登录即注册**：首次登录自动创建用户，无独立注册流程
- 验证码有效期约 5 分钟，有发送频率限制
- 登录成功返回 `login_token`，前端持久化存储，后续请求通过 Header 携带
- 数据落点：`s_user`（用户主表）、`s_user_login`（登录记录）、`s_sms_logs`（短信记录）

### 1.2 浏览信息

```
首页（分类Tab / Banner / 搜索 / 信息流）
  → 分类列表页（子分类切换 / 条件筛选 / 排序）
    → 信息详情页（完整内容 / 联系方式 / 评论 / 相关推荐）
```

- **首页**：一级分类 Tab、Banner 广告、热门分类、推荐信息流、搜索入口
- **分类列表页**：支持子分类切换、地区/价格等条件筛选、排序；同时展示信息卡片和商家卡片
- **信息详情页**：完整内容展示（标题、图片、描述、动态属性字段）、发布者信息、联系方式（聊一聊/打电话）、评论区、相关推荐
- **商家详情页**：从推荐商家区或商家卡片进入，展示商家介绍、联系电话、该商家发布的所有信息

### 1.3 发布信息（个人身份）

```
选择一级分类 → 选择二级分类 → 选择"个人"身份
  → 填写发布表单（动态字段 + 图片上传）
    → 配置增值项（置顶/推荐，可选）
      → 支付（如产生费用）→ 发布成功（信息进入待审）
```

**详细说明**：

| 步骤 | 内容 | 备注 |
|------|------|------|
| 分类选择 | 一级→二级，数据来源 `s_fenlei_category` | 不同分类对应不同模板和字段 |
| 身份选择 | 选择"个人"（选"推荐商家"则跳转商家认证） | — |
| 填写表单 | 系统字段按模板渲染，在"详细描述"位置插入自定义字段 | 图片通过七牛云直传 |
| 增值项 | 置顶时长、推荐套餐、显示天数 | 可选，产生额外费用 |
| 支付 | 微信/支付宝扫码支付 | 免费额度内无此步骤 |
| 发布成功 | 信息状态变为"待审"(`status=2`) | 可选：再发一条 / 返回首页 |

**费用规则**：
- 每月有免费发布额度（如 3 条），超额需付费
- 费用按"分类 × 用户组"维度配置
- 支付介质：现金（RMB）、金币、iOS 虚拟币

### 1.4 发表评论

```
进入信息详情页 → 点击评论区 → 填写评论内容 → 提交
  → 评论进入待审状态 → 运营审核通过后展示
```

- 需登录后操作
- 评论状态：待审(`status=1`) → 通过(`status=2`) / 不通过(`status=3`)
- 数据落点：`s_fenlei_comment`

### 1.5 用户中心

| 功能 | 说明 |
|------|------|
| 我的发布 | 按状态分 Tab（全部/通过/待审/拒绝/待支付/已失效/已删除），支持编辑、下架、刷新、续期、成交标记 |
| 我的评论 | 查看已发表的评论，可跳转到所属信息 |
| 我的收藏 | 查看收藏的信息，支持取消收藏 |
| 个人资料 | 修改昵称、头像等基本信息 |

### 1.6 信息状态流转（用户视角）

```
发布 → 待审(status=2) ──通过──→ 已发布(status=1) ──到期/下架──→ 已失效(status=4)
                       ──拒绝──→ 已拒绝(status=3)，可编辑后重新提交
                                 已失效 ──刷新──→ 已发布(status=1)
```

## 二、商家（未认证/认证中/认证过期/认证被拒）

> 商家是指发起了商家认证但尚未获得有效认证状态的用户。拥有普通用户的全部能力，以下仅列出差异部分。

### 2.1 商家认证申请

**入口**：
- 发布信息时选择"推荐商家"身份 → 系统检测未认证 → 跳转认证页
- 用户中心"我的认证"主动发起

**认证流程**：

```
查看认证须知弹窗（会员说明 + 客服电话）→ 点击"知道了"
  → 填写认证信息 → 提交 → 等待运营审核
```

**认证信息项**：

| 字段 | 说明 |
|------|------|
| 商家 Logo | 图片上传（七牛云直传） |
| 商家名称 | 文本输入 |
| 所属分类 | 下拉选择 |
| 店铺特色标签 | 多标签输入 |
| 联系电话 | 手机号 |
| 企业地址 | 地址输入 |
| 企业名称 | 文本输入 |
| 企业介绍 | 富文本/长文本 |

- 数据落点：`s_stores`（商家主表）、`s_group_user`（用户组认证记录，group_id=1025）

**认证状态流转**：

```
提交认证 → 待审
  ──通过──→ 已认证(status=1)，有效期通常 365 天 → 见"认证商家"角色
  ──拒绝──→ 已拒绝(status=2)，显示驳回原因 → 可编辑后重新提交
  ──过期──→ 可申请续期认证
```

### 2.2 发布限制

- 选择"推荐商家"身份发布时，系统检查认证状态：
  - **审核中**：可能需等待通过后才能以商家身份发布
  - **已拒绝/过期**：跳转重认证页面
  - **未认证**：跳转认证页面
- 仍可以"个人"身份正常发布

## 三、认证商家

> 认证商家是商家认证已通过且在有效期内的用户（`s_group_user.status=1`）。拥有普通用户的全部能力，以下仅列出额外权益和差异。

### 3.1 以"推荐商家"身份发布

```
选择一级分类 → 选择二级分类 → 选择"推荐商家"身份
  → 系统校验认证状态（已通过）→ 直接进入发布表单
    → 填写表单 → 增值项 → 支付 → 发布成功
```

- 发布流程与个人身份基本一致，但身份标记不同
- 发布的信息会关联到商家主体（`s_stores`），在前台展示商家标识

### 3.2 展示特权

| 特权 | 说明 |
|------|------|
| 首页推荐商家区 | 认证商家优先出现在首页"推荐商家"模块 |
| 分类列表商家卡片 | 分类列表中优先展示认证商家卡片 |
| 商家详情页 | 拥有独立的商家详情页（Logo、介绍、联系方式、在售信息列表） |
| 认证标识 | 信息卡片和详情页展示"认证商家"标签 |

### 3.3 可能的套餐权益

- 认证商家可能获得额外的免费发布次数、刷新次数、置顶额度等
- 具体权益取决于运营配置的套餐内容

### 3.4 商家管理

| 功能 | 说明 |
|------|------|
| 认证状态查看 | 查看当前认证状态、有效期 |
| 商家资料编辑 | 修改商家名称、Logo、介绍等（修改后可能需重新审核） |
| 续期认证 | 认证过期前可申请续期 |
| 重认证 | 认证过期或被拒绝后重新提交，保留之前的资料可编辑 |

### 3.5 套餐购买与使用

```
进入套餐购买页 → 查看可用套餐列表 → 选择套餐 → 支付
  → 获得套餐权益（免费发布次数 / 刷新次数 / 置顶天数等）
    → 发布时自动消耗套餐权益
```

- 套餐按"分类"维度关联（全分类或指定分类）
- 权益有有效期，过期未使用的权益作废
- 数据落点：`s_fenlei_package_user`（持有实例）、`s_fenlei_package_log`（权益变更记录）

## 四、运营人员

### 4.1 后台登录

```
输入账号 + 密码 → 登录验证 → 建立会话
```

- 独立于前台用户系统，数据表 `s_admin_manager`
- 登录记录写入 `s_admin_manager_log`

### 4.2 信息审核工作台

这是运营人员最核心的工作台，按信息状态分为多个页签：

#### A. 待审页（status=2）— 以审核为主

| 操作 | 流程 | 结果 |
|------|------|------|
| **通过** | 点击"通过" → 弹窗可选：在分类首页显示 / 通知作者 → 确认 | `status=1`，记录 `pass_at` |
| **拒绝** | 点击"拒绝" → 弹窗：选择预设理由或输入自定义理由、退款金额、退套餐、通知作者 → 确认 | `status=3`，记录 `deny_reason`，费用原路退回 |
| **编辑** | 打开编辑表单修改内容 → 提交 | 保持待审状态 |

#### B. 通过页（status=1）— 以运营管理为主

| 操作 | 说明 |
|------|------|
| **拒绝** | 已通过的信息可重新拒绝下架 |
| **编辑** | 修改内容；若触发敏感词则自动转为待审 |
| **刷新** | 让信息重新出现在列表顶部 |
| **加热** | 通过马甲互动系统提升信息热度 |
| **转发** | 自动转发信息到其他渠道 |

#### C. 拒绝页（status=3）— 以恢复或清理为主

| 操作 | 说明 |
|------|------|
| **通过** | 恢复拒绝信息为通过状态 |
| **删除** | 逻辑删除（`status=5`） |

#### D. 待支付页（is_pay=2）— 支付异常处理

| 操作 | 说明 |
|------|------|
| **通过** | 仅改变信息审核状态，不同步修改外部订单状态（有黄色警示） |

> 信息审核状态与支付订单状态是独立的两个状态机，防止账对不齐。

#### E. 已失效页（status=4）

| 操作 | 说明 |
|------|------|
| **刷新** | 重新激活信息 |
| **编辑** | 修改信息内容 |
| **删除** | 逻辑删除 |

#### F. 已删除页（status=5）— 仅供审计

- 记录删除人信息（用户自删 / 管理员删除）
- 不可恢复操作

#### 筛选与排序

- 筛选条件：关键词、信息 ID、用户名、联系电话、分类、来源（APP/小程序/PC/后台）、发布时间
- 通过页额外筛选：置顶类型、刷新类型、分类首页状态

### 4.3 评论审核工作台

```
评论列表（按状态分 Tab：待审/通过/不通过）
  → 筛选（用户名/用户ID/信息ID/评论关键词）
    → 单条操作：通过 / 不通过 / 查看详情
```

**双审核入口**：
1. **评论列表页**：独立的评论审核工作台
2. **信息详情弹窗**：审核信息时，在弹窗底部同步审核该信息下的所有评论

### 4.4 审核理由库管理

```
分类设置 → 审核操作理由 → 新增/编辑/删除/排序
```

- 维护预设拒绝理由列表，供审核拒绝弹窗下拉选择
- 支持拖拽排序
- 数据表：`s_fenlei_audit_reason`（id, content, sort, status）

### 4.5 分类管理

#### A. 分类列表

- 维护分类树（一级/二级分类）
- 设置分类启用状态、排序

#### B. 单分类基础设置

| 配置项 | 说明 |
|------|------|
| 分类名称/图标/排序/标签色 | 基本信息 |
| 发布须知 | 用户发布前展示的提示文案 |
| 模板选择 | 8 种模板（图文分享、电话突出、价格突出、招聘、拼车等），决定系统字段组合 |

#### C. 单分类字段设置

- **系统字段**：由模板决定（如价格突出模板包含 title, address, pictures, price, detail, tags, linkman, mobile）
- **自定义字段**：运营可添加额外字段
- 可调整字段名称、字数限制、是否必填、排序

#### D. 分类费用与权限配置

按"分类 × 用户组"维度联合控制：

| 费用类型 | 配置项 |
|------|------|
| 发布费用 | 每天发布总量限制、发布间隔、免费额度、超额收费 |
| 刷新费用 | 登录赠送、首发赠送、阶梯购买 |
| 置顶费用 | 发布时/发布后选择置顶，按时长收费 |
| 热门推广 | 单次申请付费 |

#### E. 全局设置

- 展示设置：首页统计数据、图标显示、推荐商家、信息流样式（单图/图文/瀑布流）
- 发布与搜索：发布样式、搜索优先级、新用户指引、发布公告
- 内容与联系：过期信息展示、阅读量设置、防骚扰、手机号查看限制

### 4.6 套餐管理

#### 套餐配置

| 配置项 | 说明 |
|------|------|
| 关联分类 | 全分类 / 指定分类 |
| 套餐名称 / 优先级 | 基本信息与排序 |
| 有效期 | 天数 |
| 限购次数 | 0=不限 |
| 价格 | 现金 / iOS 虚拟币 |
| 套餐内容 | 免费发布次数、刷新次数、置顶天数、简历库查看次数、自定义内容 |

#### 套餐上线流程

```
配置套餐 → 打开"前端显示"开关 → 点击"套餐发布" → 前端可购买
```

- 数据表：`s_fenlei_package`（套餐定义）

### 4.7 用户管理

#### A. 用户列表与详情

- 查看：用户 ID、用户名、昵称、头像、手机号、注册时间、最后活跃时间
- 资产：金币、虚拟币、余额
- 用户组：所属组列表、认证状态、有效期
- 发布记录：该用户发布的分类信息列表

#### B. 商家认证审核

```
待审认证列表 → 查看商家资料 → 通过 / 拒绝（填写驳回原因）
```

- 通过：`s_group_user.status=1`，用户获得认证商家身份
- 拒绝：`s_group_user.status=2`，用户可查看驳回原因后重新申请

#### C. 用户组管理

| 类型 | 示例 | 特性 |
|------|------|------|
| 系统组（type=0） | VIP、马甲、管理员 | 系统预设 |
| 自建组/标签（type=1） | 热心评论员、美食达人 | 运营自定义 |
| 企业认证组（type=2） | 分类信息商家 | `auth=1` 需审核，`skip_verify=1` 发布免审 |

#### D. 封禁管理

| 操作 | 参数 |
|------|------|
| 禁言 | `days >= 1`（天数），`days >= 10000`（永久） |
| 解禁 | `days = 0` |

- 记录操作者、原因、时间，数据表 `s_forbid`

### 5.2 收费体系全景

```
┌──────────────────────────────────────────────┐
│              收费场景                          │
├──────────────────────────────────────────────┤
│                                              │
│  发布费用 ←── 免费额度内免费 / 超额按次收费     │
│  置顶费用 ←── 按时长收费（发布时/发布后可选）    │
│  刷新费用 ←── 登录赠送 / 首发赠送 / 阶梯购买    │
│  热门推广 ←── 单次申请付费                      │
│  套餐购买 ←── 一次性付费获得多项权益包            │
│                                              │
│  支付介质：现金(RMB) / 金币 / iOS虚拟币         │
│  退款场景：审核拒绝时退发布费+置顶费+套餐权益     │
│                                              │
│  费用维度：分类 × 用户组（普通/商家/认证商家）    │
│                                              │
└──────────────────────────────────────────────┘
```

## 六、关键业务规则速查

| # | 规则 | 说明 |
|---|------|------|
| 1 | 登录即注册 | 手机号+验证码，无独立注册流程 |
| 2 | 双身份体系 | "个人"和"推荐商家"是两种独立的发布身份 |
| 3 | 审核与支付分离 | `status`（审核状态）和 `is_pay`（支付状态）是两个独立状态机 |
| 4 | 动态字段插入规则 | 系统字段按序渲染，在"详细描述"位置插入全部自定义字段 |
| 5 | 编辑触发重审 | 通过态信息编辑时触发敏感词则自动回到待审 |
| 6 | 拒绝自动退款 | 审核拒绝时发布费/置顶费原路退回，套餐权益退还 |
| 7 | 评论双入口审核 | 评论列表独立审核 + 信息详情弹窗内联审核 |
| 8 | 费用多维配置 | 按"分类 × 用户组"联合维度配置，支持多支付介质 |
| 9 | 套餐与单次并存 | 免费额度、套餐权益、单次付费三种计费方式共存 |
| 10 | 操作全留痕 | 审核、编辑、删除等所有后台操作写入操作日志 |
